## Cheryl messy spouse-approval verdict

**PASS — with two real gaps caught:**

### GAP 1: POSTPONED notifies Key (NEW REQUIREMENT)

Currently `bot-handoff-notifier.ts` treats POSTPONED as silent. But POSTPONED-spouse-approval leads are WARM PAUSES, not dead leads. Key should be notified so he can:
1. See in CRM that they're warm-paused (visible status)
2. Decide whether to manually follow up a week later if customer doesn't return

**Fix:** POSTPONED with `spouse_approval_needed` reason should send Key a softer notification ("Warm pause: Cheryl awaiting spouse approval, see CRM. No action needed unless you want to ping in a week.")

### GAP 2: POSTPONED should be RESUMABLE, not terminal

Currently POSTPONED is terminal (per state-machine.js `terminal: true`). When Cheryl returns 4 days later saying "Bill is on board, where were we", the bot would route her to NEEDS_CALLBACK (the universal escape for any inbound to a terminal state).

That's WRONG behavior. POSTPONED is a soft-pause; customer return should RESUME the flow at the state we paused from.

**Fix:** Make POSTPONED non-terminal. Add `paused_state` column to contacts that captures the state we paused FROM (for spouse-approval-needed). On inbound while bot_state=POSTPONED, transition to paused_state. The phraser intent on resume: "welcome back + re-ask the question we paused on."

This is a real architectural fix, worth doing before deploy.
