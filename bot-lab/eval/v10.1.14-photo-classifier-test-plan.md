## What got updated in v10.1.14

`bot-lab/agents/photo-classifier-agent.md` — added a `MAIN BREAKER DETECTION VISUAL RULES` section (rule 2a) with:

- **Primary signals**: single double-pole at top, "MAIN" label, higher amperage rating (100/125/150/200/400A printed), external main on meter (MLO panels)
- **Secondary signals**: panel size (20-42 slots = main, 6-12 = sub), wire feeder thickness
- **Absence signals**: all branch breakers same size, no double-pole at top, "subpanel/workshop" labels
- **Confidence calibration**: ≥0.85 high (silent accept), 0.5-0.85 moderate (ask customer to verify), <0.5 low (ask for clearer pic)
- **Cautious behavior baked in**: "Maya is NOT an electrician. Classifier surfaces visual facts; bot asks customer to confirm if uncertain."

Output schema additions:
- `is_main_panel_likely: bool | "unsure"` (was just bool — added "unsure" tier)
- `main_breaker_confidence: 0.0-1.0`
- `main_breaker_visual_signals: []` (debug aid)

## How to run the test once Maya is live

1. Wait for ≥10 customer panel photos to accumulate via `bot-photo-classifier` (typically 3-5 days post-pilot at expected lead volume).
2. Pull each photo URL from `qualification_data.panel_photo_url` in `contacts`.
3. Re-run each through `bot-photo-classifier` (same endpoint, no behavioral difference — just for re-verification with the v10.1.14 prompt).
4. For each verdict, Key manually reviews the photo and labels:
   - Did classifier correctly identify whether a main breaker is visible?
   - Was confidence calibration appropriate? (high confidence on clear cases, "unsure" on ambiguous)
   - Did the bot ASK the customer to verify when confidence was moderate? (CONFIRM_MAIN_BREAKER state firing)
   - Were any sub-panels falsely accepted as mains?
5. Tune prompt language for any failure modes:
   - **False positives** (calling something a main when it isn't — e.g., 60A subfeed mis-identified): tighten "PRIMARY signals" — emphasize that the main is typically the SINGLE LARGEST device, not just any double-pole.
   - **False negatives** (saying "no main" when there is one but partially obscured): adjust to lean "unsure" rather than "false" when top of panel is cut off.
   - **Overconfidence on partial views**: if confidence ≥0.85 but breaker handles aren't clearly readable, the prompt is sandbagging — add explicit visual-evidence requirements.

## Acceptance criteria pre-experiment-monitor decision

Before declaring the photo classifier "production-ready":

- 10+ real panel photos reviewed
- ≥80% correct main-breaker identification (no false positives — false negatives are recoverable via CONFIRM_MAIN_BREAKER)
- Zero confident-but-wrong ratings (anything wrong should have been "unsure")
- Hazardous panel detection (Zinsco/FPE) caught correctly when present (likely 1-2 of 10 in SC housing stock)
