# Phase 06-01: Webhook Route (Wave 1)

## Overview
Create the webhook receiver endpoint that handles all 7 Zapier trigger types with shared-secret auth, rate limiting, idempotent upserts, event logging, and auto-archive on completion.

## Tasks

### T1: Create server/routes/webhook.js

**File:** `/Users/keygoodson/Desktop/CLAUDE/permit-app/server/routes/webhook.js`

**What it does:**
- Rate limit: 50 req/min (separate from global 100/min limiter in index.js)
- Shared-secret auth via `x-webhook-secret` header vs `WEBHOOK_SECRET` env var → 403 on mismatch
- `findPerson(phone, email)` — phone normalization (strip non-digits, LIKE match last 10), then email fallback
- `upsertPersonAtStage(name, phone, email, targetStage, source)` — creates or finds person; only advances stage (never regresses)
- `logWebhook(eventType, payloadSummary, personId)` — inserts into `webhook_log` table (already in schema.sql)
- 7 event_type cases:
  1. `quote_form_submitted` → stage 1
  2. `quo_sms_replied` → stage 2
  3. `dubsado_project_status_updated` → stage 3
  4. `dubsado_new_lead` → stage 2
  5. `dubsado_new_job` → stage 4
  6. `dubsado_contract_signed` → stage 4
  7. `dubsado_payment_received` → stage 9 + `runAutoArchive()`
- Unknown event_type → 400
- Missing event_type → 400
- All errors → 500 with error logged

**Commit:** `feat(06-01): server/routes/webhook.js — all 7 event types, auth, idempotent`
