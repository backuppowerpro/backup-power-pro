# Plan 02-03: Jurisdictions Routes + Register All + Verify

## Goal
Create jurisdiction CRUD routes, register all routes in index.js, and verify every endpoint with curl.

## Requirements Covered
- JURI-09: 7 pre-seeded jurisdictions accessible via GET
- JURI-10: Seed idempotent (from 02-01, verified here)
- JURI-11: Passwords stored and returned as plaintext
- AUTO-13: webhook_log table present (schema from 02-01)
- ARCH-05: Analytics endpoints include all people (foundation for Phase 7)

## Tasks

### T1: Create server/routes/jurisdictions.js
- `GET /api/jurisdictions` — all jurisdictions ordered by name ASC
- `GET /api/jurisdictions/:id` — single jurisdiction
- `POST /api/jurisdictions` — create
- `PATCH /api/jurisdictions/:id` — partial update
- `DELETE /api/jurisdictions/:id` — hard delete

### T2: Register all routes in server/index.js
- Import and mount peopleRouter at `/api/people`
- Import and mount jurisdictionsRouter at `/api/jurisdictions`
- Import and mount eventsRouter at `/api/events`
- Mounted BEFORE health check (good practice: feature routes before static)

### T3: Verify all endpoints with curl
Tests:
1. `GET /api/health` → `{"status":"ok"}`
2. `GET /api/jurisdictions` → array of 7
3. `POST /api/people` with name + phone → 201 with id
4. `GET /api/people` → array with new person
5. `PATCH /api/people/1` with stage:2 → person with stage:2
6. `GET /api/events?person_id=1` → stage event from 1 → 2

## Success Criteria
- [ ] All 6 curl tests pass
- [ ] Server starts clean, no import errors
- [ ] 7 jurisdictions in GET /api/jurisdictions response
- [ ] Stage transition event visible in GET /api/events
