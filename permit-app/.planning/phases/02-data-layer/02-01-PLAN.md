# Plan 02-01: Database Foundation

## Goal
Install better-sqlite3, create the DB singleton, apply the full 4-table schema on startup, and seed 7 jurisdictions idempotently.

## Requirements Covered
- ARCH-04: `is_archived` + `archive_reason` columns on people
- ARCH-11: `archive_reason` CHECK constraint
- JURI-09: 7 jurisdictions pre-loaded at first run
- JURI-10: Seed is idempotent
- JURI-11: Passwords stored as plaintext in SQLite
- AUTO-13: `webhook_log` table created
- ANAL-08: Analytics derived from `people` and `stage_events` (tables established here)
- PIPE-14: `stage_updated_at` column stores timestamp for time-in-stage computation

## Tasks

### T1: Install better-sqlite3
- Run `npm install better-sqlite3` in `server/`
- Adds `"better-sqlite3": "^11.0.0"` to dependencies

### T2: Create server/db.js
- better-sqlite3 singleton
- WAL mode enabled
- Foreign keys enforced

### T3: Create server/schema.sql and server/migrate.js
- 4 tables: `people`, `stage_events`, `jurisdictions`, `webhook_log`
- `runMigrations()` exported from migrate.js
- Called in server/index.js on startup

### T4: Create server/seed.js and wire into index.js
- `seedJurisdictions()` checks count before inserting
- 7 jurisdictions inserted in a transaction
- Called after `runMigrations()` in index.js

## Success Criteria
- [ ] `permit-manager.db` is created on disk on server start
- [ ] All 4 tables exist in the DB
- [ ] 7 jurisdictions are present after first run
- [ ] Re-running seed does not duplicate jurisdictions
- [ ] Server starts without errors after changes
