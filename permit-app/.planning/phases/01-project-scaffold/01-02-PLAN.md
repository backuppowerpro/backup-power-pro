# Plan 01-02: Vite + React + Tailwind

## Goal
Scaffold the client directory with Vite 6, React 19, Tailwind CSS v4 (Vite plugin, not PostCSS), dark class forced on `<html>`, and all runtime client dependencies installed.

## must_haves
- `client/package.json` lists all required dependencies with correct versions
- `client/vite.config.js` uses `@tailwindcss/vite` plugin and proxies `/api` to `localhost:3001`
- `client/index.html` has `class="dark"` on `<html>` and `bg-slate-950` on `<body>`
- `client/src/index.css` uses `@import "tailwindcss"` and defines the `dark` custom variant
- `npm install` inside `client/` completes without errors

<task id="01-02-T4">
<action>
Create the `client/src/` directory, then create `client/src/index.css` with the following exact content:

```css
@import "tailwindcss";

@custom-variant dark (&:where(.dark, .dark *));
```

These two lines are the entire CSS configuration for Tailwind v4. The `@import "tailwindcss"` replaces the old PostCSS setup. The `@custom-variant dark` line enables class-based dark mode — any element inside a `.dark` ancestor will activate `dark:` utility variants.

Do NOT add `@tailwind base`, `@tailwind components`, or `@tailwind utilities` — those are v3 syntax and will error in v4.
</action>
