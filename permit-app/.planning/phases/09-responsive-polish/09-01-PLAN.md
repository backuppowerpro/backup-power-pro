# Phase 9: Responsive Polish — Plan

## Goal
Add skeleton loading states, toast feedback for mutations in PersonCard, audit for accidental light-mode colors, and ensure mobile touch targets meet minimum 36px.

## Tasks

### T1: Skeleton loading states
- **Pipeline.jsx**: Replace plain text loading with 3 animated skeleton rows (h-16 cards).
- **Analytics.jsx**: Replace plain text loading with skeleton matching the analytics layout (preset pills, summary cards, chart area).

### T2: Toast feedback on stage mutations in PersonCard
- Import `useToast` from `./Toast`
- Add `onSuccess` handler to the mutation that:
  - Shows `'Archived'` when `updated.is_archived` is true
  - Shows `'Moved to <StageName>'` when stage changed
  - Shows `'Restored'` otherwise
- Add `onError` handler showing `'Failed to update'` toast (error type)

### T3: Dark mode audit
- Search for accidental light-mode classes (`bg-white`, `text-black`, `text-gray-900`, `bg-gray-100`, `bg-gray-50`) without a `dark:` prefix.
- Fix any found. App is dark-only so these should not appear.

### T4: Mobile touch target audit
- PersonCard action buttons use `p-1.5` (~36px with icon). Add `min-h-[36px] min-w-[36px] flex items-center justify-center` to all action buttons (ChevronRight, ↕ move menu, Pencil, Archive/Restore).

### T5: Final build verify
