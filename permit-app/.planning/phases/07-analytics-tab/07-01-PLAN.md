# Plan 07-01: Analytics API Endpoint

## Wave: 1
## Status: Pending

## Overview
Create the backend analytics endpoint that serves stage counts, funnel data, conversion rates, and events over time.

## Tasks

### T1: Create server/routes/analytics.js
- New file: `/Users/keygoodson/Desktop/CLAUDE/permit-app/server/routes/analytics.js`
- GET /api/analytics?from=YYYY-MM-DD&to=YYYY-MM-DD
- Queries:
  - `stageCounts`: current count per stage from `people` table (all time + active split)
  - `funnelCounts`: distinct people who reached each stage via `stage_events` in date range
  - `conversionRates`: computed stage-to-stage conversion percentages (stages 1→9)
  - `eventsOverTime`: weekly new leads (to_stage=1, no from_stage) + completions (to_stage=9), grouped by strftime('%Y-%W')
  - `summary`: totalPeople, totalActive, totalCompleted
- Commit: `feat(07-01): server/routes/analytics.js — stage counts, funnel, conversion rates`

### T2: Register analytics route in server/index.js
- Add import: `import analyticsRouter from './routes/analytics.js'`
- Add route: `app.use('/api/analytics', analyticsRouter)`
- Commit: `feat(07-01): register analytics route`

## Dependencies
- `server/db.js` (already exists)
- `stage_events` table (added in Phase 02/03)
- `people` table with `is_archived` and `archive_reason` columns

## Notes
- Date defaults: from='2000-01-01', to='2099-12-31' for "All Time" queries
- Conversion rate loops from stage 1→2 through 8→9 (8 transitions total)
- `from_stage IS NULL` on stage_events identifies initial entry (no prior stage)
