## Rules for future smart work

1. **Keep the base familiar.** Don't replace a list with a novel UI just to add AI. Layer underneath.
2. **No visible AI chrome.** No robot emoji, no "powered by Sparky" — the intelligence should feel like the list just knows.
3. **Scaffold first, algorithm second.** Land the visual before wiring the real model so the UI contract is stable.
4. **One chip per row max.** Multiple stacked badges defeat the at-a-glance principle.
5. **Tone palette.** gold = peak interest. red = overdue/stuck/urgent. green = on track/paid. navy = info. purple = permit/process. muted = default/low-priority.
6. **Regex > LLM > nothing.** Start with a cheap regex heuristic. Upgrade to an LLM pass only when the heuristic clearly fails a real case.
