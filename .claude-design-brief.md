# Claude Design — Ready-to-build Brief

Tool is reusable again **2026-04-24** onward. Below is the queued design work, ordered by impact. Each item names the current state, the desired improvement, and the constraint.

## Ground rules

- **Design system is locked.** Minesweeper Brutalist + Cybertruck + 8-bit. Zero border-radius, hard 3–4px bevels, `#c0c0c0` silver / `#3d3d3d` dark, VT323 for headlines, Inter for body, stepped `steps(4)` transitions. Tokens live in `crm/v2/tokens.css`. Full reference: `wiki/CRM/Style Guide.md`.
- **Customer-facing pages are off-limits.** `proposal.html`, `invoice.html`, `index.html`, `get-quote.html`, landing pages, ad creative → keep their current polished marketing design. This brief only touches the CRM and internal tooling.

---

## Tier 1 — CRM Quick List / Briefing / Pipeline polish

**Problem:** The new Quick List default is functional but visually flat — all sections look the same height and weight, empty sections go silent. Pipeline column headers truncate (`PERMIT SUBMIT…`).

**Design asks:**
1. Quick List section headers — stronger visual rhythm between REPLY NOW (gold), TODAY (green LCD), THIS WEEK (navy), PERMITS IN FLIGHT (purple), STUCK QUOTES (red). Maybe a thin left-bar stripe tinted to match the section.
2. Empty-section hero for the WAITING chip on LIST view and the all-empty Quick List state. "CLEAR DESK" pixel-art style ASCII graphic.
3. Pipeline column headers — abbreviate systematically (PERMIT SUBMIT → SUBMIT, READY TO PAY → PAY, READY TO PRINT → PRINT) and/or rotate labels 90° on narrow columns so they stay readable down to ~70px.
4. Pipeline card "phone fallback" rows (e.g. `···8307`) — use a monospaced 8-bit font variant and dotted border to signal "no name on file yet".

## Tier 2 — Install Brief modal

**Problem:** The modal consolidates everything (name/when/where/deal/last-words) but the visual hierarchy is flat — it reads more like a form printout than an actionable "grab your keys and go" brief.

**Design asks:**
1. Hero row at top — big pixel LCD-style `INSTALL @ 10:00 AM MON · 2 DAYS PAST` with a status dot (on-time / late / today / tomorrow).
2. Portal button more prominent — make it look like a hardware push-button with tactile bevel.
3. "Last words" quote card — show in a notepad-style framed block with monospace font, more room.
4. Add a `GO →` call-to-action at the bottom. Drops into maps navigation to the address. Green LCD background to match the aesthetic.

## Tier 3 — Finance KPI cards

**Problem:** "00 INSTALLS THIS WEEK", "$12,001 OUTSTANDING", "05 OVERDUE" — static text, no affordance that they're drill-through targets. Key can see the number but can't click to see the list.

**Design asks:**
1. KPI cards should look tactile-pressable (raised bevel → pressed bevel on hover).
2. Tap a KPI → drawer slides up from bottom with the filtered list.
3. Small chevron (▸) in the corner signals the drill-through.

## Tier 4 — Quick Quote add-on chips

**Problem:** Four buttons (Cord / Surge / POM / Permit) with subtle on/off states. POM unlabeled — Key's team might forget what it stands for.

**Design asks:**
1. Expand abbreviations (POM → "Peace of Mind Plan") or add a hover tooltip with plain-English description.
2. Clearer on/off visual — maybe tint the chip gold when on, gray when off, with a mini LCD `+ $375` price adder when on.

## Tier 5 — Mobile CRM polish

**Problem:** Dark mode has subtle contrast gaps (row separators invisible, stage chip text low-contrast). Broken Street View thumbnails are filtered now but the fallback avatar octagon could be more visually distinctive.

**Design asks:**
1. Dark mode token audit — pick a pair of row-divider + stage-chip-text tokens that sit at AA contrast against the card bg.
2. Avatar octagon for no-photo leads — add a small pixel-style "??" or "∅" icon inside so phone-fallback rows read as "no name yet" at a glance.

## Tier 6 — Command palette affordance

**Problem:** `⌘K` is keyboard-only. No discoverable UI trigger.

**Design asks:**
1. Small search icon in the top bar (between dark-mode toggle and + new lead) with `⌘K` hint.
2. Palette empty-state with "Try: contacts · bookings · `settings`" preset chips.

## Tier 7 — Agents Inbox surface

**Problem:** Sparky chat and Agents inbox currently live in the same right-side panel and compete for space. Chat is default; agent notifications get buried.

**Design asks:**
1. Split the right panel vertically — top 40% is Agents Inbox (scrollable feed of Alex-tagged notifications), bottom 60% is Sparky chat.
2. Each agent notification should be a tactile card with an ACTION button (e.g. "Review photo", "Qualify lead", "Escalate").

## Tier 8 — Proposal / Invoice templates (low priority)

**Problem:** These are customer-facing and off-limits per the lock. Skip for now — track separately if the design-system debate opens for them.

## How to use this brief in Claude Design

1. Open Claude Design.
2. Start a new project keyed to an existing export from `crm/v2/` (so the design language is already loaded).
3. Pick a Tier from above — each one has enough spec to run as a single design sprint.
4. Export back to `crm/v2/` as JSX, same pattern as the original build.
5. Wire data in `app.jsx` (already the pattern for every existing component).

Keep tokens untouched unless we're intentionally evolving the system — in which case, update `wiki/CRM/Style Guide.md` first, then tokens, then components.
